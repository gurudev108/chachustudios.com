## 1. What was `/usr/local/srs/objs/nginx/html`?

In your Docker command, you map:
`-v /Users/shiningankit/Workspace/Github/chachustudios.com/srv/tv:/usr/local/srs/objs/nginx/html`

* **Inside the Docker container:** SRS (Simple Realtime Server) comes with an embedded Nginx server to distribute HTTP-based streaming formats like **HLS (`.m3u8`)** or **HTTP-FLV**. The default directory where it serves these web files is `/usr/local/srs/objs/nginx/html`.
* **What it does for you:** Your local folder `srv/tv` is masquerading as SRS's web server root. When FFmpeg pushes an RTMP stream to SRS, SRS can convert it on-the-fly to HLS and save the playlist/segments directly into that folder. This is exactly how `tv.chachustudios.com` accesses and plays the live video files via the Cloudflared tunnel!

## 2. Streamlining the Vision: Browser vs. App

> **Recommendation:** **Start with the Browser (Web App).** Building a custom iOS/macOS app adds a massive layer of complexity with Apple Developer accounts, TestFlight, and AVKit rendering. Since you already have `tv.chachustudios.com` routed via Cloudflare, turning that web page into a rich, interactive **React or Next.js Web App** is much faster. It will run flawlessly on both Safari (iOS) and Chrome (macOS).

To achieve the overlays (timers, breathing visualizers):

1. **Do not burn text into the video using FFmpeg.** It's heavy on your M2 CPU and not interactive.
2. **Layer text/animations on top of the video.** Play the stream in a standard `<video>` tag, and use absolute HTML/CSS positioning to overlay reactive React components (a ticking SVG timer, a breathing pulse animation).
